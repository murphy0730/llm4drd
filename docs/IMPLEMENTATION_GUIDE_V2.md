# LLM4DRD 前台 V2 实施指南

## 概述

这是一份完整的前台升级实施指南，包括新设计的激活方式、功能说明和未来扩展计划。

---

## 第一章：快速开始

### 1.1 激活新前台

新的前台设计包含在以下文件中：

```
frontend/
├── index_v2.html        # 新的主页面
├── app_v2.css           # 新的样式设计
└── app_v2.js            # 新的交互框架
```

**方式 1：替换原有前端（完全升级）**

```bash
cd frontend/
# 备份原版本
mv index.html index_backup.html
mv app.css app_backup.css
mv app.js app_backup.js

# 启用新版本
mv index_v2.html index.html
mv app_v2.css app.css
mv app_v2.js app.js

# 重新启动服务
uvicorn llm4drd_platform.api.server:app --reload --port 8000
```

**方式 2：灰度发布（两个版本共存）**

保持原有 index.html，在 server.py 中添加路由：

```python
@app.get("/v2")
async def index_v2():
    f = os.path.join(FRONT, "index_v2.html")
    return FileResponse(f) if os.path.exists(f) else {"msg": "V2 not available"}
```

然后访问 `http://localhost:8000/v2` 使用新前台。

### 1.2 基本操作流程（首次使用）

1. **打开应用** → 进入"场景库"
2. **新建场景** → 选择"快速生成"或"导入 Excel"
3. **加载场景** → 自动进入"智能仪表板"
4. **进入工作台** → 逐步完成 5 个步骤
5. **启动优化** → 查看候选方案
6. **导出结果** → 生成决策文档

## 第三章：用户角色与使用场景

### 3.1 领导/决策者（5 分钟汇报）

**进入路径：** 应用加载 → 选择场景 → 进入**智能仪表板**

**关键信息：**
- 问题规模  
- 关键瓶颈与预期收益
- 最优方案与推荐度
- 与现有规则的对标
- AI 背书说明

**操作：** 
1. 查看 KPI 卡片了解问题规模
2. 看洞察区的"结构分析、方案对标、智能建议"
3. 查看方案对比表的前 3 名方案
4. 点 [导出决策文档] 获得可演示的 PDF

**时间：** 3-5 分钟

---

### 3.2 计划员/工艺工程师（30 分钟问题配置）

**进入路径：** 应用加载 → **新建场景**

**关键操作：**
1. 导入 Excel 或快速生成实例
2. 进入工作台 → 第 1-2 步
3. 维护订单、工序、机器、停机、在制状态
4. 完成约束校准后，可进入仿真验证

**关键页面：**
- 新建场景 → 导入/生成选择
- 工作台 → 第 1 步（问题设计）
- 工作台 → 第 2 步（约束校准）
  - 停机维护
  - 班次管理
  - 工装与人员

**时间：** 20-40 分钟（取决于数据复杂度）

---

### 3.3 优化工程师（2 小时深度分析）

**进入路径：** 应用加载 → 选择场景 → **工作台**

**关键流程：**
1. 完成工作台前 3 步（问题设计、约束校准、结构与仿真）
2. 进入第 4 步（多目标优化）
   - 配置优化参数
   - 启动混合 NSGA3-ALNS
   - 实时监控优化进度
3. 进入第 5 步（方案评审）
   - 对比候选方案
   - 分析每个方案的机器占用、人员匹配、工装复用
   - 选择最优方案

4. 在**决策** → **洞察中心**进行深度分析
   - 结构分析：关键路径、瓶颈工序、共享资源
   - 资源分析：机器负荷、工装复用、人员技能匹配
   - 瓶颈分析：识别制约因素

**时间：** 60-120 分钟

## 第四章：核心页面详解

### 4.1 智能仪表板 (Dashboard)

**URL：** `#/page-dashboard` （导航自动跳转）

**5 个核心区域：**

#### 1. 问题概览（KPI 卡片）
- 订单数、工序数（问题规模）
- 机器类数、工装类数（资源分布）
- 优化目标个数（决策重点）
- 完成进度（工作流位置）

**交互：** 每个卡片可点击查看详情

#### 2. 关键洞察（3 列卡片）
- **结构分析**：瓶颈工序、关键路径、共享资源瓶颈
- **方案对标**：现有规则结果 vs 历史最优方案
- **智能建议**：预估改进空间（延期↓、利用率↑、成本↓）

**交互：** 每个卡片有 [查看详情] 按钮

#### 3. 候选方案对比（表格）
- 前 5-10 个 Pareto 方案
- 关键指标：延期、利用率、调度复杂度
- 推荐度星级
- 快速操作：查看甘特、导出、对比

**交互：** 
- 单击行高亮对比
- 右键菜单导出单个方案
- [详细方案评审] 进入专门页面

#### 4. 实现路径与甘特预览
- 当前最优方案的甘特图
- 基线对比 (ATC 规则)
- 资源负荷分布

**交互：**
- 悬停显示工序详情
- [展开] 进入完整甘特图编辑器
- [导出时间表] 获得生产计划

#### 5. 快速操作区
- [启动优化] → 进入工作台第 4 步
- [查看全部方案] → 方案库
- [导出决策文档] → 生成 PDF 汇报

### 4.2 工作台（Workflow）

**URL：** `#/page-workflow`

**特点：**
- 5 步清晰导航条
- 按需加载工作区内容
- 参数、结果、细节在一屏显示

#### Step 1: 问题设计
- 实例生成参数选择
- 或导入已有实例
- 目标：确认订单、工序、资源基本信息

#### Step 2: 约束校准
- 订单维护（优先级、交期）
- 工序维护（前置关系、机器、工装、人员）
- 机器班次（class-based scheduling）
- 停机管理（计划/非计划）
- 初始在制状态

#### Step 3: 结构与仿真
- **三个子任务并行：**
  1. 构建异构图（点 [构建图] 按钮）
  2. 选择规则运行仿真（ATC/EDD/SPT 等）
  3. 查看仿真结果的甘特图和 KPI

#### Step 4: 多目标优化
- 选择优化目标（1-5 个）
- 选择优化策略（快速/均衡/深度）
- 启动混合 NSGA3-ALNS
- 实时进度条
- 自动保存结果

#### Step 5: 方案评审
- 列出所有候选方案
- 对比关键指标
- 读取 AI 的方案推荐理由
- 最终选定方案
- [导出结果] 生成执行计划

## 第五章：数据流与后端集成

### 5.1 API 调用流程图

```
前端 UI 层 (app_v2.js)
    ↓ (api 模块)
API 请求/响应 (api 对象)
    ↑↓ (HTTP)
后端 FastAPI (server.py)
    ↓ (业务逻辑)
核心层 (models.py, simulator.py, rules.py)
    ↓
优化层 (hybrid_nsga3_alns.py, pareto.py, exact.py)
    ↓
结果数据
    ↑ (JSON)
前端状态 (app.instanceDetails, app.optimizeResult 等)
    ↓
UI 渲染
```

### 5.2 关键 API 端点（完全兼容）

所有现有 API 保持不变，新前台通过以下主要端点：

| 端点 | 方法 | 新前台用途 |
|-----|------|---------|
| `/api/instance/generate` | POST | 场景库、工作台 Step 1 |
| `/api/instance/details` | GET | 仪表板、场景信息栏 |
| `/api/instance/import-excel` | POST | 新建场景 → 导入 Excel |
| `/api/graph/build` | POST | 工作台 Step 3 → 构建图 |
| `/api/graph/meta` | GET | 洞察中心 → 结构分析 |
| `/api/simulate` | POST | 工作台 Step 3 → 运行仿真 |
| `/api/optimize/hybrid` | POST | 工作台 Step 4 → 启动优化 |
| `/api/optimize/{task_id}` | GET | 进度轮询 |
| `/api/llm-config` | GET/POST | 系统 → 大模型连接 |

### 5.3 状态管理架构

```javascript
app = {
  currentScene: { ... },          // 当前场景
  instanceDetails: { ... },       // 实例详情（来自 /instance/details）
  graphMeta: { ... },             // 图元数据（来自 /graph/meta）
  simResult: { ... },             // 仿真结果（来自 /simulate）
  optimizeResult: { ... },        // 最新优化结果
  optimizeHistory: [ ... ],       // 历史优化列表
  llmConfig: { ... },             // LLM 配置
  workflowProgress: { ... },      // 工作台进度追踪
}
```

### 5.4 缓存策略

- **场景列表**：应用启动时加载，支持手动刷新
- **实例详情**：选择场景时加载，编辑后重新获取
- **图数据**：构建图后缓存，支持重新构建
- **仿真结果**：按规则规则缓存，支持追加规则
- **优化结果**：优化完成后缓存全部 Pareto 前沿

**刷新策略：** 
- 顶部 [🔄 刷新] 按钮 → 刷新当前场景的所有数据
- 各工作区 [刷新] 按钮 → 局部刷新

## 第八章：故障排查

### 8.1 常见问题

**Q: 新前台加载后页面为空**
A: 检查浏览器控制台是否有 JS 错误。确保 `/api` 端点可访问。

**Q: 场景库显示不了场景**
A: 当前使用模拟数据，正式部署需要连接后端场景管理接口。

**Q: 点击按钮没有反应**
A: 检查事件委托是否正确绑定。查看 `ui.bindPageEvents()` 是否被调用。

**Q: API 请求超时**
A: 增加 `CONFIG.TIMEOUT`，或联系后端检查长时间运行的算法。

### 8.2 性能优化

- **减少重新渲染**：使用 innerHTML 而非频繁 appendChild
- **图片懒加载**：甘特图、资源视图支持虚拟化
- **数据分页**：大型场景的方案库显示前 100 个
- **索引 DB 缓存**：本地缓存场景和优化历史
