混合框架与精确求解：广搜、精修、标尺

NSGA-III 铺广度、ALNS 挖深度、CP-SAT 定标准——三者如何在一条流水线上分工。

找矿队的分工

把整套优化想成一支找矿队。矿区（参数空间）辽阔，精细勘探（完整仿真）昂贵。聪明的做法是分三层力度：

🛰️
卫星遥感（广搜）

NSGA-III + 近似评估，快速扫过整片矿区，圈出富矿带。

⛏️
钻探队（精修）

对晋级的富矿带用完整仿真 + ALNS 精细勘探。

🔬
实验室化验（标尺）

CP-SAT 精确求解，为质量定一个可信基准。

两个"评分裁判"：近似 vs 精确

混合框架的省钱关键，是备了两台评分器——广搜阶段用便宜的近似评估器初筛海量候选，只有晋级的少数精英才享受昂贵的完整仿真。

⚡

近似评估

简化排程，快一个量级。广搜阶段成百上千次调用都靠它。

🎯

精确仿真

模块 1 那台完整解码器，含资源日历、故障。只给晋级精英用。

💰
计算预算的艺术

这就是"广撒网用便宜工具、收网时才上贵工具"。把有限的仿真算力,精准砸在最可能是好解的候选上——工程里典型的预算分配思维。

大白话 · 广搜 → 晋级

阶段一（广搜）：一代代进化候选权重。

每代交叉变异出一批后代候选。

用近似评估器并行快速给它们打分（便宜、能海量跑）。

父代 + 后代合并。

用 NSGA-III 的 select_survivors（模块 4）择优——参考点保持前沿多样。

阶段二（晋级）：从广搜积累的候选池里，挑出最值得深挖的精英。

这些精英随后进入精确仿真 + ALNS（模块 5）局部精修，产出最终前沿。

标尺：CP-SAT 精确求解

启发式算法再聪明也不保证"最优"。所以系统请出 CP-SAT 精确求解器：把排产问题原原本本翻译成数学模型，求真正的最优（或给出可信的边界），当作衡量启发式解好坏的标尺。

CODE · optimization/exact.py · solve() 
for op in decision_ops:
    duration = max(1, int(round(op.work_remaining * scale)))
    start = model.NewIntVar(0, horizon, f"s_{op.id}")
    end   = model.NewIntVar(0, horizon, f"e_{op.id}")
    interval = model.NewIntervalVar(start, duration, end, f"i_{op.id}")
# 同一台机器上的所有工序区间，两两不许重叠
model.AddNoOverlap(machine_intervals[resource_id])
大白话 · 建模

为每道待决策工序建三个变量：开工时刻、完工时刻。

加工时长用 scale（×60）把小时换成整数，CP-SAT 只吃整数。

start / end 被限制在 [0, horizon] 这个时间范围内。

区间变量把开工-时长-完工绑成一个对象。

核心约束：同一台机器上所有区间 AddNoOverlap——一台机同一时刻只能干一道工序。

（多台可选机器时还会用布尔选择器 + AddExactlyOne 选一台。）

🐢
精确 = 慢，所以只当标尺

CP-SAT 在两万工序上会爆炸，通常只在小规模实例上求最优，用来验证"启发式解离最优有多近"。日常大实例排产，还是靠混合框架的启发式产出。
