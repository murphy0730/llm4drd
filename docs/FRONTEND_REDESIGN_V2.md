# LLM4DRD 前台重设计方案 V2
## 面向B端展示 + 工业级操作的大幅升级方案

---

## 第一部分：设计理念与版本对比

### 问题诊断
当前前台存在的问题：
1. **信息架构单薄**：5个标签页面，左右切换，没有真正的"发现"路径
2. **展示与操作混杂**：配置参数和展示结果在同一页面竞争注意力
3. **面向工程师**：大量低级配置，不适合给领导做汇报
4. **缺乏主线**：没有清晰的"问题→求解→方案→决策"的业务流程表现
5. **视觉层级不足**：所有信息平等展示，重点不突出

### 重设计从三个维度展开

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| **页面组织** | 5个线性标签 | 多维导航：快速决策路径 + 深层工作台 |
| **信息展示** | 配置与结果混在一起 | 仪表板（汇报） + 工作台（操作）分离 |
| **业务流程** | 技术流程导向（配置→图→仿真→优化→LLM） | 业务流程导向（问题→洞察→方案→评审→决策） |
| **视觉风格** | 工程工作台 | 智能决策中枢 + 专业操作间 |
| **用户人群** | 以技术人员为主 | 同时服务领导、计划员、技术人员 |

### 2.2 四大核心模块（替代 5 个标签页）

#### 模块 1：问题与场景管理面板

**路径：** 左导航 → 场景

虽然仍然支持所有配置功能，但改成了引导式、步骤化的设计：

```
[新建场景] [导入Excel] [快速生成] [从模板]

当前场景：Demo_20250327
├─ 问题规模：10单 / 45工序 / 3类机器 / 2类工装
├─ 实例状态：已校准（订单/工序/停机/在制状态）
└─ [查看详情] [编辑主数据] [导出模板]

场景列表：
  1. Demo_20250327  ✓ 完成配置  |  时间：2025-03-27 10:30
  2. Demo_20250326  · 工作中     |  时间：2025-03-26 14:20
  3. Production_Q1  · 初始化     |  时间：2025-03-20 08:15
```

**特点：**
- 卡片式、历史记录式，摆脱单一"配置页"
- 一键进入详细编辑模式
- 快速洞察当前实例状态，无需高度滚动

## 第三部分：交互流程示例

### 场景 A：领导做汇报（5分钟）

1. 打开系统 → 左导航选 **决策**
2. 自动加载智能仪表板
3. 展示"问题规模、关键瓶颈、最优方案、预期收益"
4. 对标"现有规则 vs Pareto 最好方案"
5. 导出"决策文档" with AI 说明

---

### 场景 B：计划员配置问题（30分钟）

1. 左导航 **场景** → 新建场景
2. 导入 Excel 或快速生成
3. 进入 **工作台**
4. 步骤 1-2：问题设计、约束校准
5. 在任何一步都能点 **运行** 看即时结果
6. 完成后推到 **化学审**

---

### 场景 C：优化工程师调参与分析（2小时）

1. 左导航 **工作台** → 完成步骤 1-3
2. 左导航 **优化与方案** → 配置参数
3. 查看实时 Pareto 前沿（边运行边看）
4. 在 **洞察中心** 对比多个方案的"机器占用、人员技能匹配、工装复用"
5. 在 **方案评审** 团队评估每个候选方案
6. 最后在 **决策** 中获取 AI 的推荐理由

## 第五部分：技术实现建议

### 5.1 前端框架改进

不强制换框架，在 Vanilla JS + 现有 CSS 基础上：

1. **模块化组件**：
   - `<dashboard-card>` - 仪表板卡片
   - `<work-panel>` - 工作台工作区
   - `<data-table>` - 表格编辑器
   - `<timeline-chart>` - 甘特图

2. **路由与状态管理**：
   - 简单的 URL hash 路由：`#/scene/{sceneId}/dashboard`
   - 本地 IndexedDB 缓存场景与最近优化结果

3. **响应式布局**：
   - PC 优先（1280px+），左侧导航固定
   - 平板支持（缩小导航，扩展内容区）

### 5.2 确保后端兼容性

所有现有接口保持不变，新前端作为完全替代品：

- `POST /api/instance/generate` ← 场景生成
- `POST /api/simulate` ← 工作台仿真
- `POST /api/optimize/hybrid` ← 优化启动
- `GET /api/optimize/{task_id}` ← 实时进度
- 等等（所有现有接口）

## 总结

这个重设计方案的核心理念是：

> **从"工程工作台" → "智能决策中枢"**
>
> - 前台分离：**汇报路径**（仪表板）vs **操作路径**（工作台）
> - 业务优先：从技术流程 → 业务流程
> - 层级清晰：领导看结论 → 计划员看计划 → 工程师看细节
> - 一物多表：同一个数据，按场景展示（表单 vs 表格 vs 图表）

既满足领导要求（一屏看尽、决策支持、AI 背书），也满足操作人员要求（步骤清晰、参数灵活、结果即时）。
