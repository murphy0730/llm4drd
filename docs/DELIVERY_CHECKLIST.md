## 🚀 快速启动（3 种方式）

### 方式 1：完全替换（推荐正式部署）

```bash
cd frontend/

# 1. 备份原版本（保险起见）
mv index.html index_backup_v1.html
mv app.css app_backup_v1.css
mv app.js app_backup_v1.js

# 2. 启用新版本
mv index_v2.html index.html
mv app_v2.css app.css
mv app_v2.js app.js

# 3. 重启服务
pkill -f "uvicorn llm4drd_platform"
uvicorn llm4drd_platform.api.server:app --reload --port 8000

# 4. 访问应用
# 打开浏览器：http://localhost:8000
```

**完成！** 用户现在使用 V2 前台。

## 🎯 使用场景举例

### 场景 1：领导做 5 分钟汇报

```
1. 打开应用
2. 应用自动进入仪表板（如已加载场景）
3. 看到：
   - 问题规模（订单数、工序数）
   - 关键瓶颈与改进空间
   - 3 个候选方案的对比
   - AI 的推荐理由
4. 点 [导出决策文档] 获得 PDF
5. 关闭应用
⏱️ 总耗时：5分钟
```

---

### 场景 2：计划员配置一个新问题

```
1. 打开应用 → 场景库
2. 点 [新建场景]
3. 选择 [导入 Excel] → 选择本月的订单文件
4. 系统导入，自动进入工作台 Step 1
5. 验证订单、任务、工序信息
6. 进入 Step 2：维护班次、停机、在制状态
7. 完成后，点 [下一步] 进入 Step 3
8. 在 Step 3 中运行规则仿真验证可行性
9. 保存场景
⏱️ 总耗时：20-30分钟
```

---

### 场景 3：优化工程师深度分析

```
1. 打开应用 → 选择场景
2. 进入仪表板快速浏览
3. 进入工作台 → 完成 Step 1-3（问题与仿真）
4. 进入 Step 4：启动混合优化
   - 选择优化目标：最小延期、最大利用率
   - 选择策略：深度（180s）
   - 运行
5. 进入 Step 5：方案评审
   - 对比前 5 个方案
   - 查看每个方案的甘特图与资源占用
   - 读取 AI 的方案评价
   - 选择最优方案
6. 进入决策 → 洞察中心 → 做深度分析
   - 结构分析：关键路径、瓶颈工序
   - 资源分析：机器占用、工装复用
   - 瓶颈分析：建议投资方向
7. 导出完整分析报告
⏱️ 总耗时：60-120分钟
```

## 📞 常见问题快速解答

### Q: V2 有没有破坏性改动？
**A:** 完全没有。V2 100% 兼容 V1 的所有后端 API 和数据格式。

### Q: 用户需要重新学习吗？
**A:** 学习成本很低。基本操作与 V1 相同，新增功能（仪表板、工作台引导）提供更好的体验。推荐 15 分钟的简短培训。

### Q: 我能不能只用 V2 的某些功能？
**A:** 可以。V2 的每个功能都是独立的。可以只用仪表板（汇报），或只用工作台（配置），或完整使用。

### Q: V2 对网络连接有要求吗？
**A:** 前端代码是静态 HTML/CSS/JS，对网络要求低。后端 API 仍然需要正常连接，这与 V1 相同。

### Q: 我能修改仪表板的颜色吗？
**A:** 可以。编辑 `app_v2.css` 中的 CSS 变量（`:root { --color-primary: ...}`），所有颜色自动更新（支持主题切换）。

### Q: 后端需要改吗？
**A:** 不需要。V2 前端可直接与现有后端 API 配合，无需任何改动。可选的未来改进（场景管理、导出）不影响现有功能。
