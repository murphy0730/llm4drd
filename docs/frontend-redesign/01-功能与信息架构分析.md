# LLM4DRD 前端重构 · 功能与信息架构分析

> 分析对象：`frontend/index_v2.html` + `app_v2.js`（约 6000 行）+ `app_v2.css` / `design-system.css`
> 结论：现有前端**功能完整度很高**，问题集中在**信息架构与视觉引导**——导航是"页面清单"而非"流程向导"，用户需要自行记忆工作流顺序与前置依赖。

## 二、现有信息架构的 5 个问题

### 问题 1：导航是"页面清单"，不是"流程向导"
侧栏导航 `01 数据导入 / 02 规则仿真 / 03 优化求解 / 04 方案评审` 虽然有编号，但：
- 编号没有**状态**（已完成/进行中/被阻断），用户不知道上一步做完没有
- 「工作概览」与主流程 4 步平级，但它其实是"全局仪表盘"，层级关系错位
- 未加载实例时 `requires-scene` 项只是变灰，**没有解释为什么灰、怎么解锁**

### 问题 2：功能入口分散且层级混乱
- 「文件导出」藏在「系统状态」页的 Tab 里，但它是评审完成后的**主流程终点动作**
- 「图谱构建」放在「数据导入」页，但图谱是独立的数据准备阶段，认知上值得独立入口
- 「大模型连接」是 AI 评审的前置条件，但导航上两者没有任何关联提示

### 问题 3：工作流状态反馈依赖用户记忆
- 顶栏只有实例数据统计（订单/任务/工序…），没有**流程进度**
- 唯一体现流程的组件是「工作概览」页里的 5 步进度条，但它只在一个页面里存在，离开该页就看不到了
- 仿真失败、校验未通过等阻断状态，只在各自页面内提示，没有全局可见的"当前卡在哪一步"

### 问题 4：页面内部信息密度不均
- 「方案评审 · 方案库」单页塞了：决策横幅 + 方案池工具 + 对比表 + 利用率 + 甘特图，纵向滚动极长
- 「优化求解」把"参数配置表单"和"运行状态/结果"堆在同一流式布局里，运行时用户要在长页面里上下滚动看进度
- 「数据导入」页把上传、校验、图谱三件事纵向排列，视觉重点分散

### 问题 5：视觉层级平淡
- 全部卡片统一的 1px 边框 + 无阴影，重要状态（如优化运行中、校验失败）仅靠淡色背景区分，不够醒目
- 主按钮、次按钮、幽灵按钮在多页面混用，没有明确的"每页一个主动作"原则
- 设计 token 已有完整定义（`--primary/--accent/--success/--info`），但实际用色保守，强调色 `--accent` 几乎没用到

## 三、重构方案

### 3.1 新信息架构：以工作流为骨架

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏：品牌 + 流程进度条（5步，常驻）+ 场景切换 + LLM 状态    │
├──────────┬──────────────────────────────────────────────────┤
│          │  页面头部：当前步骤 + 前置检查 + 下一步引导       │
│  侧栏    ├──────────────────────────────────────────────────┤
│  导航    │                                                  │
│  (分组)  │              页面主内容区                        │
│          │                                                  │
│  ·流程   │  （每页一个主动作，卡片分级清晰）                │
│  ·分析   │                                                  │
│  ·系统   │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

**侧栏导航新分组：**

| 分组 | 项目 | 状态灯逻辑 |
|---|---|---|
| **调度流程** | 01 数据导入 | 绿=已导入且校验通过 / 黄=已导入有警告 / 红=校验失败 / 灰=未导入 |
| | 02 图谱构建（从数据导入拆出） | 绿=已构建 / 灰=未构建 |
| | 03 规则仿真 | 绿=已仿真可行 / 黄=仿真不完整 / 灰=未运行 |
| | 04 优化求解 | 蓝脉冲=运行中 / 绿=已完成 / 灰=未运行 |
| | 05 方案评审 | 显示已选方案数徽标（如 `2/4`） |
| **分析** | 工作概览 | — |
| **系统** | 导出与交付（从 Tab 提升为独立页） | — |
| | 大模型连接 | 绿=已连接 / 黄=未配置 |
| | 系统状态 | — |

### 3.2 顶栏流程进度条（全局常驻）

把现有「工作概览」页里的 5 步进度条提升为**全局组件**，固定顶栏下方：
- 每步显示：步骤名 + 状态（✓ 完成 / ● 当前 / ○ 待开始 / ⚠ 阻断）
- 可点击跳转到对应步骤（前置不满足时给出 toast 提示原因）
- 当前步骤高亮 + 进度连接线动画

### 3.3 页面级改进

| 页面 | 改进 |
|---|---|
| 数据导入 | 上传区居中放大为 Hero 区；校验结果改为"错误/警告/通过"三档大色块摘要 + 明细折叠；图谱构建移出 |
| 图谱构建（新页） | 构建状态卡 + 图谱元数据 + 可视化画布，独立页面有充足空间 |
| 规则仿真 | 规则选择改为卡片式（6 个规则带一句话业务解释）；不可行告警提到页面顶部通栏 |
| 优化求解 | 左侧参数配置（sticky）+ 右侧运行状态与结果，运行时进度区用大号进度环 + 阶段指示（近似广搜→精确精修） |
| 方案评审 | 方案库内部改为"左：方案列表（卡片式可选中）/ 右：对比详情 Tab（指标对比/利用率/甘特）"，替代超长纵向滚动 |
| 导出与交付（新页） | 模板下载 / CSV 导出 / 方案 Excel 导出，作为流程终点动作独立呈现 |
| 工作概览 | 保留 Hero + KPI + 三列详情，流程进度条移除（已提升到全局） |

### 3.4 视觉语言调整

- **保留浅色基调与主色**：`--primary #2f6feb` 延续，深色模式 token 保留备用
- **激活强调色**：`--accent #c2620a` 用于"当前步骤"标识、预算推荐提示等需要用户注意的中性强调
- **卡片分级**：一级卡片（主操作区）用 8px 圆角 + 轻阴影；二级信息卡用 6px 圆角 + 1px 边框；状态卡（运行中/失败）用 2px 彩色左边条
- **每页一个主动作**：页面头部右侧固定一个主按钮（如"运行仿真"/"启动优化"/"导出方案"），其余操作降级为次按钮或幽灵按钮
- **数据可视化规范**：甘特图机器行配色按机器类型取色板，利用率 ≥85% 绿 / 60–85% 蓝 / <60% 灰

### 3.5 保留的既有约定（不改）

- 方案显示名：「方案一 / 方案二…」，不出现 `S-xxxxxxxx` solution_id
- 评审勾选上限 4 个，与 AI 评审共用勾选集
- `.is-selected` / `.is-best` 状态样式语义
- 设计 token 变量名（`--primary` 等），便于后续映射回 `design-system.css`
- 所有后端 API 与数据结构（本设计稿为纯前端表现层重构）

## 四、设计稿说明

配套文件：`02-设计稿.html`（单文件、可直接浏览器打开，v2 已按用户反馈修订）

### v2 修订记录（对照用户 6 点反馈）

1. **工作流去重**：流程表达收敛到顶栏下方唯一的 5 步进度条；侧栏移除步骤编号（仅保留页面入口 + 状态灯）；删除每页的大标题 Header 区（eyebrow + H2 + 描述），页面只保留一行前置检查条 + 内容，不再三处重复。
2. **图谱可视化**：基于现有「可交互有向异构图」Tab 重构——保留焦点邻域/全关系视图、适配/重置/缩放/全屏、订单过滤+跳转、节点搜索、节点层级/关系层级筛选、面包屑统计、画布统计、右侧节点详情与关系分解；工具栏/筛选/面包屑压缩为紧凑行式布局，**画布区域显著放大**（整页宽 + min-height 560px），右侧详情栏收窄至 320px。
3. **仿真甘特**：保留原 vis-timeline 甘特样式与**全部筛选能力**（订单多选、分组模式[按订单层级/按机器资源]、机器类型多选、仅含停机、机器号搜索、工序状态、搜索、时间窗、分页），仅把菜单整理为 3 行紧凑筛选条。
4. **优化目标排列**：目标从 2×N 大卡片网格改为**紧凑 chip 流式网格**（checkbox 内嵌），目标很多时自然换行、不占纵向空间，方向（min/max）用小字标注。
5. **方案评审 · 方案库**：改为对比表结构（方案多 → 表体纵向滚动 + 表头吸顶）；**列配置保留**（下拉面板勾选，12 个 KPI 可选，默认显示 3 个主目标 + 2 个常用 KPI 共 5 列，主目标列带标记）；**操作列**重做为一行三枚小按钮「◎ 详情 / ✦ AI 评审 / ⬇ 导出」，「◎ 详情」与下方利用率表、甘特联动（当前方案名高亮同步）。
6. **利用率 + 甘特回归**：「机器分类利用率对比 · 当前方案」保留并强化——按机器类型 × D1..Dn 逐日利用率矩阵，过载（≥90%）红、偏低（<60%）黄、行内最佳绿；下方补回**方案排程甘特**，样式与筛选能力和仿真甘特完全一致（同一组件），随「◎ 详情」切换方案。

### 页面清单
9 个页面：数据导入 / 图谱构建 / 规则仿真 / 优化求解 / 方案评审（3 Tab）/ 工作概览 / 导出与交付 / 大模型连接 / 系统状态。关键交互均可用：导航切换、流程步骤点击、方案勾选（含 4 个上限提示）、列配置开关、操作列三动作、利用率/甘特联动、图谱筛选 chip、多选筛选下拉、AI 对话演示。模拟数据基于真实字段（6 订单 / 12 任务 / 86 工序 / 9 机器 / 6 启发式规则 / 5 候选方案 / 12 个 KPI 指标）。
