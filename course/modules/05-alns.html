ALNS：破坏、修复、再破坏

拿一个好解反复"拆一块、重拼一块"，专治遗传算法磨不动的局部最优。

卡在半山腰怎么办

遗传算法能找到一个不错的解，但常常卡在局部最优——周围小改都变差，可它并不是最好的。ALNS 的破局思路很暴力：不小改，而是拆掉一大片重来。

🧩
拼图返工

一幅快拼好的拼图，某个区域别扭。你不会一块块微调，而是整片拆下来重拼——这就是"大邻域"。而哪种拆法、哪种拼法屡屡奏效，你会记住、下次更常用它——这就是"自适应"。

💥

破坏算子

拆掉一片派工决策，比如"延误订单相关的"、"瓶颈机器上的"、"共享工装冲突的"。

🔧

修复算子

把拆掉的部分重排，比如"按交期"、"优先主订单"、"给瓶颈削峰"。

"自适应"：让好算子出头

系统有 6 种破坏 + 5 种修复算子。每次用轮盘赌按权重挑一对。用完后，根据战绩即时调整它的权重：

CODE · optimization/alns_core.py 
def _update_operator(self, state, accepted, improved):
    state.uses += 1
    if improved:
        state.weight = min(6.0, state.weight * 1.12 + 0.08)
    elif accepted:
        state.weight = min(6.0, state.weight * 1.04 + 0.03)
    else:
        state.weight = max(0.2, state.weight * 0.94)
大白话 · 战功考核

每个算子记一笔"出场次数"。

若这次带来了改进 → 权重大涨（×1.12），下次更可能被选中。

若没改进但结果被接受了 → 权重小涨（×1.04），有功劳也有苦劳。

若被拒 → 权重打折（×0.94），但保底 0.2，永不彻底出局。

于是好用的算子自然浮上来，废柴沉下去——却都保留一线生机去探索。

关键一步：故意接受"更差"的解

ALNS 借用了模拟退火的接受准则——有时明知变差也接受，为的是翻过山头去够更高的山。

CODE · alns_core.py · refine 
delta = candidate_score - current_score
if delta <= 0 or self.rng.random() < math.exp(-delta / max(0.05, temperature)):
    accepted = True
    improved = delta < 0
# ... 每轮末尾
temperature *= 0.92
大白话 · 退火接受

delta = 新解得分 − 当前得分（这里越小越好，负的 delta 表示变好）。

变好（delta≤0）就无条件接受。

变差也别急着拒：按 e^(−delta/温度) 的概率接受它。温度越高，越敢接受烂解。

delta 只有严格变好时才算 improved（影响算子考核）。

每轮结束温度 ×0.92 慢慢降——前期大胆探索，后期收敛精修。
