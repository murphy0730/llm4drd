## 三、文档必须覆盖的内容（按章节组织，章节顺序即学习顺序）
第0章 预备知识（务必用最白话讲，这是小白的入口）
  - 什么是"调度/排产"？用"工厂做菜/医院排手术"类比。
  - 基本对象：订单、任务令、工序、机器、工艺类型、工装/人员（本项目工装=人员=0，点明即可）。
  - 工序依赖（前驱/后继）与"只能前一道做完才能做后一道"。
  - 机器可选集（eligible machines）：同一工序可由哪几台机器做。
  - 关键指标（必须每个都给定义+例子）：makespan（总完工时间）、flow time（流经时间）、
    tardiness（延误，相对交期 due date）、总完成时间、设备利用率、在制品 WIP。
  - 一个贯穿全书的"统一迷你案例"：例如 3 个订单、共 7 道工序、2 台机器的小例子，
    后面每个算法都用它演算，降低认知负担。

第1章 派工规则（Dispatching Rules）——"每当下一道机器空闲，先挑哪道工序做？"
  - 什么是派工规则、为什么需要它（实时决策 vs 全局最优的取舍）。
  - 逐个讲本项目实现的规则，每个都给：① 直觉（它偏爱哪种工序）② 评分公式（来自 core/rules.py）
    ③ 用统一案例**手算一遍**数值例子（列出 2~3 道工序的得分，比较谁先被选中）。
    规则清单：SPT（最短工时优先）、LPT、EDD（最早交期）、CR（临界比）、FIFO（先到先做）、
    MST（最小松弛）、**ATC（表观延误成本，本项目主打规则，重点讲透它的指数项与参数含义）**，
    以及业务复合规则 PRIORITY/KIT_AWARE/BOTTLENECK/COMPOSITE 的思路。
  - 规则优劣对比表（哪些擅长缩短 makespan、哪些擅长少延误）。
  - compile_rule_from_code：说明本项目允许用一段 Python 代码"自定义规则"，这是后面 LLM 进化的基础。

第2章 离散事件仿真（Discrete-Event Simulation）——"派工规则怎么被真正跑起来？"
  - 什么是离散事件仿真（对比"每秒钟 tick 一遍"的时钟驱动，讲事件驱动更快）。
  - 本项目的事件循环：一道工序做完 → 触发机器派工 → 选出下一道工序 → 机器开始加工 → 再触发……
  - 就绪队列（ready queue）与机器空闲派工：用统一案例画时间轴，标明每道工序"何时开始/结束"。
  - 仿真能算出第0章那些指标（makespan、延误等），因此仿真 = 给一套排产方案"打分"。

第3章 多目标优化与帕累托前沿（Pareto）——"又好又快还省钱，但鱼和熊掌不可兼得"
  - 为什么排产是多目标（makespan、延误、利用率常常互相打架）。
  - 什么是"非支配/帕累托最优"：用统一案例的 4~5 个候选方案，手画二维图，标出谁支配谁、
    哪些点在帕累托前沿上。
  - 目标标量化的陷阱（简单加权会丢失好解），引出"要保留一整条前沿"。

第4章 NSGA-III 多目标进化算法——"让一群方案互相进化，逼出多样的好解"
  - 进化算法母题：种群、个体、适应度、交叉、变异（用"养了一窝方案，优胜劣汰"类比）。
  - 非支配排序（fast_nondominated_sort）：分层，第一层最优秀。
  - 参考点（reference points）与小生境选择（niche）：NSGA-III 相比 NSGA-II 的核心改进——
    用参考点把种群"均匀推开"，保证很多目标时解也分布均匀。给出参考点生成的直觉 + 小例子。
  - 种群如何用仿真（第2章）做适应度评估。

第5章 ALNS 自适应大邻域搜索——"先大刀阔斧拆，再聪明地补"
  - 大邻域搜索思想：对当前解做"破坏（删掉一段）+ 修复（重新塞回去）"得到邻域解。
  - 自适应：哪些破坏/修复算子效果好就多派它上场（score 自适应）。
  - 本项目的图引导算子（来自 hybrid_nsga3_alns.py）：assembly_chain_destroy（拆装配链）、
    bottleneck_machine_destroy（专拆瓶颈机器相关工序）、shared_tooling_destroy、
    assembly_sync_repair 等——讲清"为什么用图（工序依赖）来指导拆哪、补哪"。
  - 与 NSGA-III 的混合：进化负责"广撒网找多样好解"，ALNS 负责"把每个解打磨得更优"。

第6章 CP-SAT 精确求解——"不靠猜，靠数学证明最优"
  - 精确法 vs 启发式：精确法能证最优但慢，启发式快但不保证最优。
  - 约束规划（CP）与 SAT：把"工序不重叠""前后依赖""机器同一时刻只做一道"翻译成约束。
  - 本项目实现（optimization/exact.py）：用 OR-Tools 的 CpModel，给每道工序建 start/end 整数变量，
    加 AddNoOverlap（机器不重叠）、AddExactlyOne（工序选一台机器）、Minimize(makespan)。
    用第0章迷你案例**手建一个 2 变量的小模型**示意，不必完整求解。
  - 讲清 time_limit 与"证明最优 vs 给个当前最好界"的区别。


第7章 把一切串起来——端到端排产闭环
  - 用一张流程图串起：数据(订单/工序/机器) → 图建模(knowledge/graph.py) → 求解
    （NSGA-III+ALNS 或 CP-SAT）→ 用派工规则在仿真里评估 → 选出帕累托最优排产 → 上线执行。
  - 四步骤回顾：生成（实例）→ 建模 → 求解 → 结果，呼应本项目主流程。
  - 给读者的"下一步学习路线"建议。
